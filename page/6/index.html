Python 部署（Sae）

Posted on 2015-02-05   |   In Python   |  

之前用 Sae (新浪云平台) 一直是玩 Php，学了一点 Python 之后，就发现可以部署在 Sae上。

Sae 里面的文档虽然说的比较详细，但是还是花了点时间，比 Php 稍微复杂，使用的项目为上一篇博文中的简单示例。

这里简单记录一下遇到的问题和解决方法。

1 新建一个 SAE Python Web 应用

参照[文档](http://sae.sina.com.cn/doc/python/tutorial.html#hello-world)，使用 Flask 框架，完成 Hello World 应该不是问题。

2 初始化 Sae 上的 Mysql 模块

Sae 上 Mysql 使用的管理工具是 PhpMyAdmin，可以打开这个工具，把数据库需要的 user 表建立       出来。

[![2015-02-04 12:02:29屏幕截图](http://llzz.me/wp-content/uploads/2015/02/2015-02-04-120229屏幕截图.png)](http://llzz.me/wp-content/uploads/2015/02/2015-02-04-120229屏幕截图.png)

3 添加 Peewee 模块

在上一个项目中，使用到了 Flask，Mysql，Peewee 等模块，看了 Sae 预装的模块当中是没有           Peewee 的，所以需要添加。

在添加前其实需要先安装 sae-python ：

$ git clone https://github.com/sinacloud/sae-python-dev-guide.git
$ cd sae-python-dev-guide/dev_server
$ python setup.py install
这样就安装进去了，先用 pip install flask 把小辣椒装一下。再装 Peewee 库：

saecloud install peewee
这样就安装好了。[文档查阅](http://sae.sina.com.cn/doc/python/tools.html)

4 连接 Sae MySql

需要更改 model.py 的连接数据库代码：

database = MySQLDatabase(sae.const.MYSQL_DB, **{'password': sae.const.MYSQL_PASS,\
 'user': sae.const.MYSQL_USER,\
 'host':sae.const.MYSQL_HOST, \
 'port':int(sae.const.MYSQL_PORT)})

5 问题

   AttributeError: 'module' object has no attribute 'jsonify'


可能是 flask json 包的问题，没有 jsonify ，所以更改一下：

import json
def makeReponse( ret, reason, **kwarg ):
      res = { 'ret' : ret, 'reason': reason }
      if( kwarg ):
      res.update( **kwarg )
      return json.dumps( res )

这样算是部署完成了。

Python 初入

Posted on 2015-01-29   |   In Python   |  

其实我不太清楚 Python 具体有一些什么优点，带着一点玩的心态就学习了下。

当看到官网的 The State of Python (2 vs 3) 我就觉得被坑了，到底是使用2还是3，

官方竟然是这样写的 ：

Use Python 3 if...

You don’t care.
You love Python 3.
You are indifferent towards 2 vs 3.
You don’t know which one to use.
You embrace change.
Use Python 2 if...

You love Python 2 and are saddened by the future being Python 3.
The stability requirements of your software would be improved by a language and runtime that never changes.
Software that you depend on requires it.
看完这个我也没在意到底用哪个，使用的 ubuntu 自带有 python 2.7,就直接用了。 粗略看了下基本语法，接下来完成一个**用户注册和登陆**的功能。   * * * * **基本框架&工具** [Flask ](http://dormousehole.readthedocs.org/en/latest/ "Flask") 小辣椒，微型网络开发框架 [Peewee](http://peewee.readthedocs.org/en/latest/ "Peewee") ORM MySQL 这个估计没人不知道 [Psycopg2](http://initd.org/psycopg/ "Psycopg2") 数据库接口 Mysql-workbench 数据建模工具   * **环境搭建** 1> 建立虚拟环境 Virtualenv 
$ virtualenv pythonenv
cd pythonenv/bin

pip install flask
pip install peewee
2&gt; 安装 psycopg2    （[详细请看](http://stackoverflow.com/questions/5420789/how-to-install-psycopg2-with-pip-on-python/5450183)）

sudo apt-get install libpq-dev python-dev
3&gt; 安装mysql和mysql-workbench

到此建立了一个 pythonenv 独立的虚拟环境，就可以进入项目开发了。

  • 可以安静的写代码了

    在此之前，最好先试试自己的环境是否搭建好，Flask 的一个小例子可以试一下（here）。

    随意路径下建立自己的 project（文件夹），名称就是 MyBlog ，新建一个 user.py 。

    1> 导入需要用的包

from flask import Flask,request,render_template
 from flask import json
 from model import *

app = Flask(name)

2> 定义服务端返回方式

def makeReponse( ret, reason, kwarg ):
 res = { ‘ret’ : ret, ‘reason’: reason }
 if( kwarg ):
 res.update( kwarg )
 return json.jsonify( **res )

3> 定义注册方法和登陆方法

# regist
@app.route(‘/user/regist’, methods=(‘post’,))
def registUser():
print( request.data )
req = json.loads( request.data )
account = req[‘account’]
password = req[‘pwd’]
try:
res = User.create( account=account, password = password, sex=1 )
userId = res.get_id()
return makeReponse( 0, ‘’, userId=userId )
except Exception,e:
print( str(e) )
return makeReponse( -1, str(e) )

# login
@app.route(‘/user/login’,methods=(‘POST’,) )
def loginUser():
req = json.loads( request.data )
acc = req[“account”]
pw = req[“pwd”]
try:
for info in User.select().where( User.account == acc ):
if( info.password <> pw ):
raise Exception(‘invalid password ~’)
return makeReponse( 0,’’,token=acc,userId=info.get_id() )
raise Exception(‘user not found ‘)
except Exception ,e:
print( str(e) )
return makeReponse(-1,str(e))

到此，就写完了~

Ant 自动打包 Apk

Posted on 2015-01-21   |   In Android   |  

 

曾经打包一下 Android 程序也是飞快，当听到产品需求说需要统计各个应用市场的下载量，才知道 too young too simple.

随便选了一个方式，就用 Ant 吧。

  • Ant 安装

    我直接用 apt-get 下载的 Ant，官网点 这里

    安装成功输入 ant 会出现：

Buildfile: build.xml does not exist!
 Build failed
打包工具是将编译出的 DEX 文件，资源文件以及 AndroidManifest.xml 文件（二进制格式）组合       成一个应用程序包（Apk），所以自动打包的时候可以循环迭代 Manifest.xml 里面的内容，可以         获得各种版本包名渠道的Apk文件。

  • Ant 迭代 ant-contrib-1.0b3.jar

    下载点 这里

    这里需要对这个 jar 包进行下更改，解开这个 jar 包，更改文件，

    (net/sf/antcontrib/antcontrib.propertie)

    注释：

#if=net.sf.antcontrib.logic.IfTask
因为这里的 if 语句会和 Android Sdk 里面的 if 冲突，会报错误：

if doesn't support the "condition"attribute
更改完之后重新编译成 jar 包，将这个 jar 包放到 ant 安装目录。

 

 

  • 根目录添加 ant.propertties 文件

    该文件进行 签名 版本信息等配置，主要是针对 ant 打包时需要替换的信息

key.store=my
 key.alias=com.light.anttestdemo
 key.store.password=lightlz
 key.alias.password=111111
 market_channels=Wandoujia,360,XiaoMiStore
 app_name=AntTest
 app_version=1.0.1
先将签名文件放在项目根目录下，key.store的值就是你复制进去的签名文件（.keystore后缀）

market_channels 是市场的渠道，这里主要是打包出各个市场的 Apk 让有盟能够统计出来

 

 

  • 生成 local.properties & build.xml 文件

    输入命令：

android update project --target android-20 --name AntTestDemo --path /home/light/project/workspace/AntTestDemo
  这句命令成功后，项目会出现两个文件：local.properties &amp; build.xml

（如果有引用第三方库，请按照上面的命令为该库统一生成 local.properties &amp; build.xml）

  local.properties    #location of the SDK. This is only used by Ant

sdk.dir=/usr/workspace/android/android-sdk/android-sdk-linux
 ant.dir=/usr/share/ant
 target.dir=/home/light/project/antRelease
 target.dir 是 Apk 文件输出的文件夹，当生成 Apk 的时候该目录需要一个以版本号为名称的文件          夹，所以在这个目录下再新建一个 ant.properties 里面输入的版本号为名称的文件夹

build.xml   #ant 的构建文件，类似与makefile

<property file=”ant.properties“ />

<!— 循环执行，ant-contrib 官网使用方式有介绍 —>

<taskdef resource=”net/sf/antcontrib/antcontrib.properties” >

<classpath>

<!— <pathelement location=”${ant.dir}/lib/ant-contrib-1.0b3.jar” /> —>

<pathelement location=”/usr/share/ant/ant-contrib-1.0b3.jar”/>

</classpath>

</taskdef>

<echo>Run ant-contrib-1.0b3.jar ok</echo>

<target name=”deploy”>

<foreach target=”edit_and_build” list=”${market_channels}” param=”channel” delimiter=”,”>

</foreach>

</target>

<target name=”edit_and_build”>

<echo>Run ‘${channel}’ apk</echo>

<replaceregexp

encoding=”utf-8”

file=”AndroidManifest.xml“

flags=”s”

match=’android:name=”UMENG_CHANNEL”.+android:value=”([^”]+)”‘

replace=’android:name=”UMENG_CHANNEL” android:value=”${channel}”‘/>

<property name=”out.final.file” location=”${target.dir}/${app_version}/${app_name} V${app_version}(${channel}).apk“ />

<antcall target=”clean” />

<antcall target=”release” />

</target>

这里构建文件中很容易看懂，先指定了 property file="<span style="color: #333333;">ant.properties</span>" ，在循环迭代里面的配置

replace 中会替换 AndroidManifest.xml 里面 android:name="UMENG_CHANNEL" 这一项，所         以一定要确保 AndroidManifest.xml 文件中有这个字段

迭代完之后 Apk 就会在<span style="color: #333333;"> ${target.dir}/${app_version}/${app_name} V${app_version}                          (${channel}).apk" ，所以路径一定得有 ${target.dir}/<span style="color: #008000;">${app_version}</span></span>

在project.properties中配置：proguard.config=${sdk.dir}/tools/proguard/proguard-                        android.txt:proguard-project.txt

 

 

  • 执行打包

    在此项目的路径下，执行 ant deploy ，Apk将出现在制定的路径中

    当然也可以自己写一个 .sh 做 .bat 作为批处理文件

cd /项目路径 ant deploy pause

 

Ant 只是其中一种批量打包的方式，还有apktool，maven，gradle 都可以~

Android Studio 体验

Posted on 2014-12-11   |   In Android   |  

12/08 Android Studio 1.0发布，据说是稳定版。好吧那就再下载来试试（之前下过两次，然后…）

用VPN残忍的下了一晚，感觉世界都慢了。

下完之后，按照standard安装了。

就来说说，AS到底是哪里比较帅？

首先再次来了一个HelloWorld，加载速度比之前快了很多。

改了个theme：Darcula和字体大小，纯黑的背景好有极客范，有没有！？

 

在xml查看的时候，发现竟然可以看到全部尺寸的效果，这个是eclipse没有的。帅不？

当写布局文件的时候打入<Button 按enter之后，竟然会直接给你显示出layout_height&width，再也不用自己动手再输入了！

关于快捷键，我之前是用的eclipse，所以可以直接在settings搜索keymaps可以设定。

在使用的时候遇到一个问题：SDK Manager 无法打开，直接闪退，在eclipse当中是没有问题的。

解决方案：

直接修改 Android.bat中
rem Check we have a valid Java.exe in the path.
set java_exe= C:\Program Files\Java\jdk1.7\jre\bin\java.exe
REM call lib\find_java.bat
REM if not defined java_exe goto :EOF

更改之后就ok了。

体验就到这，其实这都还不是as的优点，还有很多很帅的地方。
