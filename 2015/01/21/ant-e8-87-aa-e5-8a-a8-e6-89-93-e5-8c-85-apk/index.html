Ant 自动打包 Apk

Posted on 2015-01-21   |   In Android   |  

 

曾经打包一下 Android 程序也是飞快，当听到产品需求说需要统计各个应用市场的下载量，才知道 too young too simple.

随便选了一个方式，就用 Ant 吧。

  • Ant 安装

    我直接用 apt-get 下载的 Ant，官网点 这里

    安装成功输入 ant 会出现：

Buildfile: build.xml does not exist!
 Build failed
打包工具是将编译出的 DEX 文件，资源文件以及 AndroidManifest.xml 文件（二进制格式）组合       成一个应用程序包（Apk），所以自动打包的时候可以循环迭代 Manifest.xml 里面的内容，可以         获得各种版本包名渠道的Apk文件。

  • Ant 迭代 ant-contrib-1.0b3.jar

    下载点 这里

    这里需要对这个 jar 包进行下更改，解开这个 jar 包，更改文件，

    (net/sf/antcontrib/antcontrib.propertie)

    注释：

#if=net.sf.antcontrib.logic.IfTask
因为这里的 if 语句会和 Android Sdk 里面的 if 冲突，会报错误：

if doesn't support the "condition"attribute
更改完之后重新编译成 jar 包，将这个 jar 包放到 ant 安装目录。

 

 

  • 根目录添加 ant.propertties 文件

    该文件进行 签名 版本信息等配置，主要是针对 ant 打包时需要替换的信息

key.store=my
 key.alias=com.light.anttestdemo
 key.store.password=lightlz
 key.alias.password=111111
 market_channels=Wandoujia,360,XiaoMiStore
 app_name=AntTest
 app_version=1.0.1
先将签名文件放在项目根目录下，key.store的值就是你复制进去的签名文件（.keystore后缀）

market_channels 是市场的渠道，这里主要是打包出各个市场的 Apk 让有盟能够统计出来

 

 

  • 生成 local.properties & build.xml 文件

    输入命令：

android update project --target android-20 --name AntTestDemo --path /home/light/project/workspace/AntTestDemo
  这句命令成功后，项目会出现两个文件：local.properties &amp; build.xml

（如果有引用第三方库，请按照上面的命令为该库统一生成 local.properties &amp; build.xml）

  local.properties    #location of the SDK. This is only used by Ant

sdk.dir=/usr/workspace/android/android-sdk/android-sdk-linux
 ant.dir=/usr/share/ant
 target.dir=/home/light/project/antRelease
 target.dir 是 Apk 文件输出的文件夹，当生成 Apk 的时候该目录需要一个以版本号为名称的文件          夹，所以在这个目录下再新建一个 ant.properties 里面输入的版本号为名称的文件夹

build.xml   #ant 的构建文件，类似与makefile

<property file=”ant.properties“ />

<!— 循环执行，ant-contrib 官网使用方式有介绍 —>

<taskdef resource=”net/sf/antcontrib/antcontrib.properties” >

<classpath>

<!— <pathelement location=”${ant.dir}/lib/ant-contrib-1.0b3.jar” /> —>

<pathelement location=”/usr/share/ant/ant-contrib-1.0b3.jar”/>

</classpath>

</taskdef>

<echo>Run ant-contrib-1.0b3.jar ok</echo>

<target name=”deploy”>

<foreach target=”edit_and_build” list=”${market_channels}” param=”channel” delimiter=”,”>

</foreach>

</target>

<target name=”edit_and_build”>

<echo>Run ‘${channel}’ apk</echo>

<replaceregexp

encoding=”utf-8”

file=”AndroidManifest.xml“

flags=”s”

match=’android:name=”UMENG_CHANNEL”.+android:value=”([^”]+)”‘

replace=’android:name=”UMENG_CHANNEL” android:value=”${channel}”‘/>

<property name=”out.final.file” location=”${target.dir}/${app_version}/${app_name} V${app_version}(${channel}).apk“ />

<antcall target=”clean” />

<antcall target=”release” />

</target>

这里构建文件中很容易看懂，先指定了 property file="<span style="color: #333333;">ant.properties</span>" ，在循环迭代里面的配置

replace 中会替换 AndroidManifest.xml 里面 android:name="UMENG_CHANNEL" 这一项，所         以一定要确保 AndroidManifest.xml 文件中有这个字段

迭代完之后 Apk 就会在<span style="color: #333333;"> ${target.dir}/${app_version}/${app_name} V${app_version}                          (${channel}).apk" ，所以路径一定得有 ${target.dir}/<span style="color: #008000;">${app_version}</span></span>

在project.properties中配置：proguard.config=${sdk.dir}/tools/proguard/proguard-                        android.txt:proguard-project.txt

 

 

  • 执行打包

    在此项目的路径下，执行 ant deploy ，Apk将出现在制定的路径中

    当然也可以自己写一个 .sh 做 .bat 作为批处理文件

cd /项目路径 ant deploy pause

 

Ant 只是其中一种批量打包的方式，还有apktool，maven，gradle 都可以~
